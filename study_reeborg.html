程式邏輯的具體實踐：以 Reeborg's World 探索程式設計

「透過撰寫程式碼驅動機器人抵達終點，不僅是練習語法，更是學會如何巧妙地應用條件判斷與迴圈。」[cite: 1]

一、 學習動機與階段規劃

在掌握了 Python 的 if 條件判斷式與 while 迴圈後，我希望透過圖像化的挑戰來優化我的通用邏輯[cite: 1]。

  • 第一階段： 熟悉 Python 基礎語法與機器人基本指令[cite: 1]。
  • 第二階段： 練習基礎關卡，將指令組合成初步邏輯[cite: 1]。
  • 第三階段： 挑戰隨機起點關卡，並發展出「通用邏輯」以應對所有場景[cite: 1]。

二、 核心邏輯架構與推理思路

我發現不同關卡其實可以透過同一套邏輯達到終點[cite: 1]。我的推理思路如下：

  1. 自定義函數： 定義 turn_right() 為左轉三次[cite: 1]。
  2. 持續判斷： 使用 while not at_goal() 確保機器人在抵達前持續運作[cite: 1]。
  3. 優先順序： 我設定：右側無障礙物則右轉 → 前側無障礙物則前行 → 皆不符合則左轉[cite: 1]。

三、 困難解決與學習反思

遭遇困難： 在挑戰隨機起點關卡時，我發現機器人若在空曠處生成且缺乏牆壁做判斷，會陷入無窮迴圈[cite: 1]。

學習反思： 這次經驗讓我明白邏輯上的漏洞，雖然目前知識尚不足以完美解決「空曠定位」，但這激發了我對自動駕駛與感測定位技術的好奇[cite: 1]。

「這份暫時無法解決的 Bug，反而成了我深入學習 Python 與人工智慧的動力。」[cite: 1]
