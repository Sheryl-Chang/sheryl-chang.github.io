程式邏輯的具體實踐：以 Reeborg's World 探索程式設計

「透過撰寫程式碼驅動機器人抵達終點，不僅是練習語法，更是學會如何巧妙地應用條件判斷與迴圈。」

一、 學習動機與階段規劃

在掌握了 Python 的 if 條件判斷式與 while 迴圈後，我希望透過圖像化的挑戰來優化我的通用邏輯。

  • 第一階段： 熟悉 Python 基礎語法與機器人基本指令。
  • 第二階段： 練習基礎關卡，將指令組合成初步邏輯。
  • 第三階段： 挑戰第四關隨機起點關卡，並發展出「通用邏輯」以應對所有場景。

二、 核心邏輯架構與推理思路

我發現不同關卡其實可以透過同一套邏輯達到終點。我的推理思路如下：

  1. 自定義函數： 定義 turn_right() 為左轉三次，簡化代碼。
  2. 持續判斷： 使用 while not at_goal() 確保機器人在抵達前持續運作。
  3. 優先順序： 這是最困難的部分。我設定：右側無障礙物則右轉 → 前側無障礙物則前行 → 皆不符合則左轉。

三、 困難解決與學習反思

遭遇困難： 在挑戰「起點位置隨機生成」的關卡時，我發現如果機器人生成於空曠處、缺乏牆壁作為判斷基準，程式會陷入無窮迴圈，超出系統負荷。

學習反思： 這次經驗讓我明白邏輯上仍有漏洞。雖然目前的知識尚不足以完美解決「空曠定位」的問題，但這激發了我對自動駕駛與感測定位技術的好奇。

「這份暫時無法解決的 Bug，反而成了我深入學習 Python 與人工智慧的動力。」
